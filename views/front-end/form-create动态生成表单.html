    # 1.前言

    在最近的工作中，要实现一个需求,需要在弹窗中实现两个表单的生成，其中一个流水表的模板是固定的，业务表的模板不固定，大概有十个左右，所以这里我们不能使用固定的写法去生成表单，不然要在html界面写十个模板，再通过一个标识来切换使用不同的模板，这样属实是太麻烦了，所以我们就要通过数据来动态生成表单，下面介绍一个很好用的表单生成器form-create。

    # 2.form-create介绍

    form-create 是一个可以通过 JSON 生成具有动态渲染、数据收集、验证和提交功能的表单生成器。并且支持生成任何 Vue 组件。结合内置17种常用表单组件和自定义组件，再复杂的表单都可以轻松搞定。

    本篇文章只是简单的介绍一下在前后端不分离的情况下，下载源码到本地使用以及其中遇到的一些问题，具体npm或者cdn安装使用方式请点击form-create官网

    # 3.插件下载

    由于项目上使用的是ElementUI，所以我们这里使用elementui的版本，而且整个项目不是前后端分离，所以不能使用npm下载插件，我们要么使用cdn的方式引入，或者将插件源码下载到项目中使用。这里主要讲一下在下载插件源码的时候遇到的一些问题和使用时的一些方法。npm方式的下载就不介绍了，十分简单方便。

    # CDN引入:

    <!-- import Vue.js -->
<script src="//vuejs.org/js/vue.min.js"></script>
<!-- import stylesheet -->
<link href="https://cdn.jsdelivr.net/npm/element-ui@2.8.2/lib/theme-chalk/index.css" rel="stylesheet">
<!-- import ElementUI -->
<script defer src="https://cdn.jsdelivr.net/npm/element-ui@2.8.2/lib/index.js"></script>
<!-- import form-create -->
<script src="//unpkg.com/@form-create/element-ui/dist/form-create.min.js"></script>

    # fApi ($f)

    fApi对象上写了很多的组件方法，这里介绍几个实际中最常见到的。

    $f.formData(): 获取表单的键值对，使用场景：在我们修改或者新增完之后需要将表单里的数据传递到后端，这时候就可以将使用$f.formData()方法获取到这些数据。

    $f.fields(): 获取表单的所有字段名。

    $f.getValue(field): 获取指定字段的值

    $f.setValue(field,value): 设置字段的值

    $f.removeField(field): 删除指定字段

    $f.changeStatus(): 获取表单中的值是否发生变化状态,使用场景：在表单修改完完毕点击提交按钮后进行表单验证是否发生改变，如没有进行任何修改则不进行update更新操作。
