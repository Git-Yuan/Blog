# 实例对象

# 1.什么是对象

面对对象编程是现在主流的编程规范，它将真实世界的各种事物抽象成一个个对象，通过对象之间的分工合作，完成对真实世界的模拟。 并且对象可以复用，还有自身的继承机制，所以面对对象编程具有灵活、代码可复用、高度模块化等特点，容易维护和开发。

# (1)对象是单个实物的抽象

"万物皆对象"，一个人，一杯茶，一台电脑，什么实物都能被抽象成对象，当实物被抽象成对象，实物之间的关系就变成了对象之间的关系。

# (2)对象是一个容器，里面封装了方法和属性

属性是对象的状态，方法是对象的行为。每一个对象都有自己的属性，比如每个人都有自己的姓名和年龄等属性信息，每个人都会有自己的特长爱好，这些就是对象的方法。

# 2.构造函数

既然对象是单个实物的抽象，那么我们要如何才能生成一个对象呢？这就要用到我们的构造函数，构造函数就像是一个模板，通过这个模板，我们就可以表示一类实物的共同特征，然后生成一个对象。

典型的面向对象编程语言（比如 C++ 和 Java），都有“类”（class）这个概念。所谓“类”就是对象的模板，对象就是“类”的实例。但是，JavaScript 语言的对象体系，不是基于“类”的，而是基于构造函数（constructor）和原型链（prototype）。

JavaScript 语言使用构造函数（constructor）作为对象的模板。所谓”构造函数”，就是专门用来生成实例对象的函数。它就是对象的模板，描述实例对象的基本结构。一个构造函数，可以生成多个实例对象，这些实例对象都有相同的结构。

构造函数具有如下的几个特点：

  • 函数内部使用this关键字，代表了要生成的对象实例
  • 生成对象实例的时候，必须使用new命令
  • 构造函数的函数名第一个字母大写

# 3.生成实例对象

# 3.1 使用new命令

new命令的作用就是执行构造函数，返回一个实例对象

var People = function(){
    this.age = 22;
}
var king = new People();
king.age //22

# this关键字

在javascript中，一切皆对象，运行环境也是对象，所以this关键字返回的肯定总是一个对象，而this返回的是哪一个对象，取决于函数运行时的环境。所以要想弄明白this的指向，就需要弄清楚当前的函数运行的环境。

# 1.实质

JavaScript 语言之所以有 this 的设计，跟内存里面的数据结构有关系。

var obj = { foo:  5 };
1

上面的代码将一个对象赋值给变量obj。JavaScript 引擎会先在内存里面，生成一个对象{ foo: 5 }，然后把这个对象的内存地址赋值给变量obj。也就是说，变量obj是一个地址（reference）。后面如果要读取obj.foo，引擎先从obj拿到内存地址，然后再从该地址读出原始的对象，返回它的foo属性。

原始的对象以字典结构保存，每一个属性名都对应一个属性描述对象。举例来说，上面例子的foo属性，实际上是以下面的形式保存的。

{
  foo: {
    [[value]]: 5
    [[writable]]: true
    [[enumerable]]: true
    [[configurable]]: true
  }
}

# 对象的继承

面对对象编程很重要的一个方面就是继承，A对象通过继承B对象，就能直接拥有B对象的所有的属性和方法。而javascript与大部分的编程语言不一样，它的继承不是通过class(类)来实现继承，而是通过原型对象(prototype)。

# 1.原型对象介绍

# 1.1 构造函数的缺点

从上文中我们可以得知，通过构造函数，我们可以生成新的实例对象，这个实例对象可以继承构造函数的属性和方法。而且一个构造函数可以生成多个实例对象，但这些实例对象之间无法共享属性，这就造成了对系统资源的浪费。构造函数上的一个方法和属性，在生成实例对象的时候，每个对象都生成了各自独立的方法和属性，就等同与生成多少个实例对象，就需要生成多少个属性和方法，这样是十分消耗系统资源的。因为有的属性和方法，就应该共享。而解决这个问题的方法，就是JavaScript的原型对象。

# 1.2 prototype属性的作用

JavaScript 继承机制的设计思想就是，原型对象的所有属性和方法，都能被实例对象共享。也就是说，如果属性和方法定义在原型上，那么所有实例对象就能共享，不仅节省了内存，还体现了实例对象之间的联系。

function Animal(name) {
  this.name = name;
}
Animal.prototype.color = 'white';

var cat1 = new Animal('大毛');
var cat2 = new Animal('二毛');

cat1.color // 'white'
cat2.color // 'white'

上面代码中，构造函数的prototype属性就是实例对象的原型对象，我们为构造函数Animal的prototype属性上添加了一个color属性，构造函数的所有实例对象都共享了这一属性，而且修改原型对象上的属性，实例对象上的属性立刻就会变动，而如果单纯的修改构造函数上的值，只有在修改完之后生成的实例对象才能继承最新的值，修改之前生成的实例对象还是继承原有的未修改之前的值。

而如果实例对象和原型对象都有一个相同的属性，那实例对象就不会去原型对象上找这个属性。

其实原型对象就是定义所有实例对象都能共享的方法和属性，而实例对象可以视作从原型对象衍生出来的子对象。

# 1.3 原型链

在javascript中，每个对象都有自己的原型对象(prototype)，对象上有着很多的方法和属性，当我们想要找一个方法或者属性，会就近的查找自己本身有没有有，如果没有的话，就查找自己的原型对象，如果原型对象也没有，原型对象也有自己的原型对象，一直往后找，最后会找到对象的原型(Object.prototype),如果都没有，那就会返回undefiend，这样就形成了一条原型链。而Object.prototype也是一个对象，而只要是对象就有自己的原型对象， 通过Object.getPrototypeOf()方法，最后发现Object.prototype的原型是null，所以原型链的尽头是null。

在读取属性和方法的时候，如果对象本身和原型上都定义了相同名字的属性和方法，那么会优先的读取对象自身的属性。

# 1.4 constructor属性

prototype对象上有一个constructor属性，默认指向prototype对象所在的构造函数。constructor属性定义在原型对象上，所以可以被所有实例对象继承。 有了constructor属性之后，我们就可以得知某个实例对象是哪个构造函数产生的。而且constructor属性指向的是构造函数，说明我们可以通过constructor属性来创建实例对象。通过constructor的name属性，我们也可以知道构造函数的名称。

constructor属性表示原型对象与构造函数之间的关联关系，如果修改了原型对象，一般会同时修改constructor属性，防止引用的时候出错。

function Person(name) {
  this.name = name;
}

Person.prototype.constructor === Person // true

Person.prototype = {
  method: function () {}
};

Person.prototype.constructor === Person // false
Person.prototype.constructor === Object // true

# 1.6 构造函数的继承

让一个构造函数继承另一个构造函数，主要分三步实现

  • 1.在子类的构造函数中，调用父类的构造函数
  • 2.将子类的原型指向父类的原型
  • 3.重新指定constructor属性 EX： 父类Shape
function Shape() {
  this.x = 0;
  this.y = 0;
}

Shape.prototype.move = function (x, y) {
  this.x += x;
  this.y += y;
  console.info('Shape moved.');
};

# 1.7 多重继承

javascript不提供多重继承功能，不允许一个对象同时继承多个对象，但通过变通的方法，可以实现这个功能。

function M1() {
  this.hello = 'hello';
}

function M2() {
  this.world = 'world';
}

function S() {
  M1.call(this);
  M2.call(this);
}

// 继承 M1
S.prototype = Object.create(M1.prototype);
// 继承链上加入 M2
Object.assign(S.prototype, M2.prototype);

// 指定构造函数
S.prototype.constructor = S;

var s = new S();
s.hello // 'hello'
s.world // 'world'

# 2.封装私有变量：构造函数的写法

function StringBuilder() {
  var buffer = [];

  this.add = function (str) {
     buffer.push(str);
  };

  this.toString = function () {
    return buffer.join('');
  };

}

  add: function (str) {

  toString: function () {
    return this._buffer.join('');

};

# 3.封装私有变量：立即执行函数的写法

另一种做法是使用“立即执行函数”（Immediately-Invoked Function Expression，IIFE），将相关的属性和方法封装在一个函数作用域里面，可以达到不暴露私有成员的目的。且外部无法调用内部变量。

　var _count = 0;
　var m1 = function () {
　  //...
　};
　var m2 = function () {
　　//...
　};
　return {
　　m1 : m1,
　　m2 : m2
　};
})();

# 4.模块的放大模式

如果一个模块很大，必须分成几个部分，或者一个模块需要继承另一个模块，这时就有必要采用“放大模式”（augmentation）。

var module1 = (function (mod){
　mod.m3 = function () {
　　//...
　};
　return mod;
})(module1);

上面的代码为module1模块添加了一个新方法m3()，然后返回新的module1模块。

在浏览器环境中，模块的各个部分通常都是从网上获取的，有时无法知道哪个部分会先加载。如果采用上面的写法，第一个执行的部分有可能加载一个不存在空对象，这时就要采用"宽放大模式"（Loose augmentation）。

var module1 = (function (mod) {
　//...
　return mod;
})(window.module1 || {});

# 5.输入全局变量

独立性是模块的重要特点，模块内部最好不与程序的其他部分直接交互。 为了在模块内部调用全局变量，必须显式地将其他变量输入模块。

var module1 = (function ($, YAHOO) {
　//...
})(jQuery, YAHOO);
