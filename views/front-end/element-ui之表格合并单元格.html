    element-ui 中的表格合并行单元格

    YuanJin 2020-06-25 前端 element

    # 1.前言

    最近在工作中有一个需求，就是把一个表格的一列相同内容的单元格合并成一个单元格，如下图：

    在element官网给我们提供了一个合并单元格方法 => :span-method="objectSpanMethod"，以及一个实现的例子，但是那个例子属实是不太好理解，看了几遍还是挺懵，还得自己来上手实践一番。

    首先我们需要知道官网提供给我们的 :span-method="objectSpanMethod" 该怎么用，根据官网提示，这个方法具有四个参数{ row, column, rowIndex, columnIndex};那我们直接输出这四个参数:

    从输出的结果来看，row => 指的是当前行，column => 指的是当前列， rowIndex => 指的是当前的行号，columnIndex => 指的是当前的列号；而且表格式从0开始数的，这点很关键；除此之外我们还发现，一次输出了25条数据，从这25条数据可以看出来，它是一个一个单元格横着遍历过来的。

    接下来，我们根据官网给的例子做适当的修改，来更好的理解代码；

    • if ( columnIndex === 0 ) + 返回值 return {rowspan: 3,colspan: 1} 时

    可以看到第0列的三个单元格为一组的合并到了一起（数据不够，看的不够直观）

    • if ( columnIndex === 1 ) + 返回值 return {rowspan: 2,colspan: 1} 时

    可以看到第1列的两个单元格为一组的合并到了一起，且中间的那条是因为第二个if语句 ( rowIndex % 3 === 0 )导致的，到这里我们大概就能知道columnIndex，rowIndex 和 返回的对象 {rowspan: 3,colspan: 1} 具有什么作用了。

    columnIndex : 控制列 (从0开始)
    rowIndex : 控制行 (从0开始)
    rowspan : 要合并的单元格的个数 (上下合并)
    colspan : 要合并的列数 (colspan为0时则整个单元格消失,为1时不消失，占领一列，一般只取0或1)

    # 3.进阶阶段

    通过上面的学习，我们已经大概了解了整个 :span-method="objectSpanMethod" 的一个基础用法，接下来我们就要在我们的需求之上开始加上我们自己的条件来实现我们的需求。我们的需求也很简单，就是把一列中相邻的内容相同的单元格合并在一起，这就要我们对数据进行一个判断，最主要的就是返回值要 {rowspan;clospan}要正确，这里我们的返回值rowspan要是一列中相邻的内容相同的单元格的个数，这时候colspan为1，那我们怎么获得一个我们需要的返回值呢？这就要我们对数据tableData进行一些操作。

     var vm = new Vue({
    el: "#app",
    data() {
        return {
            spanArr:[],
            position:0,
            tableData: [{
                id: '12987122',
                name: '王小虎',
                amount1: '234',
                amount2: '3.2',
                amount3: 10
            }, {
                id: '12987123',
                name: '王小虎',
                amount1: '165',
                amount2: '4.43',
                amount3: 12
            }, {
                id: '12987124',
                name: '王小虎',
                amount1: '324',
                amount2: '1.9',
                amount3: 9
            }, {
                id: '12987125',
                name: '王大龙',
                amount1: '621',
                amount2: '2.2',
                amount3: 17
            }, {
                id: '12987126',
                name: '王大龙',
                amount1: '539',
                amount2: '4.1',
                amount3: 15
            }]
        }
    },
    mounted(){
        this.rowspan();
    },
    methods: {
        rowspan(){
            this.tableData.forEach((item,index)=> {
                if(index === 0){
                    this.spanArr.push(1);
                    this.position = 0;
                }else{
                    if(this.tableData[index].name == this.tableData[index-1].name){
                        this.spanArr[this.position] += 1;
                        this.spanArr.push(0);
                    }else{
                        this.spanArr.push(1);
                        this.position = index;
                    }
                }
            });
        },
        objectSpanMethod({ row, column, rowIndex, columnIndex }) {
            if (columnIndex === 1) {
                 var _row = this.spanArr[rowIndex];
                 var _col = _row>0?1:0;
                    return {
                        rowspan: _row,
                        colspan: _col
                    };
            }
        }
    }
})

    我们在data中新添加了一个spanArr数组，和一个position 数字0，还在methods方法中新建了一个rowspan方法，首先来看这个rowspan方法，一开始对表格数据进行一个foreach循环，之后的if( index === 0 ) 将数组push(1)，然后position为 0，这里的spanArr数组push(1)的意义就是第一个数据就是占一行，等到第二行开始，我们让第二行和第一行进行比较，比较某个键值是否相同，如果相同，则将刚开始push进去的第一个数字进行 +1 处理，之后再push(0),如果不相同则直接push(1)，然后把position的值换成索引值index，也许这样看还是很迷，有点难以理解，那我们可以直接打印出进行判断过后的rowspan数组  [3, 0, 0, 2, 0],再结合tableData的数据，其中前三个是相同的，后两个是相同的，这样我们就得到了objectSpanMethod 方法中的返回值的rowspan，而如何获得我们的colspan呢，其实我们在上面就已经测试过了colspan是合并多少列，一般我们在合并行单元格的时候，colspan要么取1，要么取0，取1代表占一列，取0代表不占列，直接没了。那什么时候取1什么时候取0呢，结合我们的rowspan来看，当我们的rowspan大于0时，说明接下来的这几行都要被合并成一行，所以colspan取1；而当rowspan等于0时，说明这一行已经被合并了，消失了，这时候colspan取0。

    最后还有几点很关键，就是我们要在获取到数据之后马上执行rowspan函数，让它生成spanArr数组。最好不要在mounted里面执行rowspan函数，因为我们页面获取数据一般都是异步操作，所以rowspan会比getData函数先执行，那就会导致整个表格界面出错，所以最好的选择就是在getData函数里面在数据获取完毕之后立马执行rowspan函数。还有一点就是有时候表格如果有分页，点击下一页之后，要将spanArr数组和position进行重置清零操作，不然原始数据保留在里面，会导致整个界面出错。
