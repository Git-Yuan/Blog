    element-ui 日期选择器范围时间限制

    YuanJin 2020-07-07 10:25:36 element-ui vue 日期选择器

    • 1.前言
    • 2.基础版本
    • 2.两个日期选择器，失效日期不能比生效日期小

    # 1.前言

      最近在用到element-ui的时间选择器来实现一个需求，其中有一个生效日期和失效日期,当然element有一个自带的选择日期的范围，你只要在单个选择框中的type属性中加上daterange或者monthrange就可以实现日期或者月份的范围选择，但是在实际开发中，遇到规定好的界面，我们就无法使用这个自带的两条范围日期选择，而element-ui也给我们留了一个自定义选项，就是picker-options属性。通过它，我们可以实现一个甚至多个日期选择器的使用范围限制。

    # 2.基础版本

    我们需要注意，picker-options是写在data中的，picker-options其中带有很多的参数，其中包括如下几个：

    • shortcuts object类型 主要用来设置快捷选项，通过传递 { text, onClick } 对象来实现
    • disabledDate 回调函数类型 设置禁用状态，参数为当前日期，要求返回 Boolean

    这边我们只需要用到disabledDate，其他的就不一一介绍了。

    • 组件代码
        <el-date-picker 
        v-model="Date" 
        type="date" 
        value-format="yyyy-MM-dd"
        placeholder="选择日期" 
        :picker-options="pickerOption">
    </el-date-picker>

    一开始我们介绍了，disabledDate的参数是当前的日期，而返回值是一个Boolean值，如果直接return false,会发现和原样没什么区别，但return true时，整个选择框处于一个无法选择的状态。由此我们可以知道，当返回值为true时，选项会处于一种无法选中的状态。而如果我们输出一下这个time参数，会发现它是循环输出了很多个时间。 由此我们大概能够推断出来，整个disabledDate里面其实做了很多次运算，当我们 return time.getTime() < Date.now(); (这里比较的是时间戳的大小) 小于当前日期的就是true，而return true是一个无法选择的状态，所以整个效果就会是今天之前的日期我们都无法选择，如下图：

    同理可知我们 return time.getTime() > Date.now(); 时，大于当前日期的选项都是不可选的；

    由此我们大概知道了整个disabledDate是如何运作的，接下来就开始写我们的需求，两个选择框，一个选择生效日期，一个选择失效日期，而失效日期是不可以比生效日期要大的。

    # 2.两个日期选择器，失效日期不能比生效日期小

    • 组件代码
        <el-date-picker 
        v-model="startDate" 
        type="date" 
        value-format="yyyy-MM-dd"
        placeholder="选择生效日期" 
        :picker-options="pickerOptionStartDate">
    </el-date-picker>
    <el-date-picker 
        v-model="endDate" 
        type="date" 
        value-format="yyyy-MM-dd"
        placeholder="选择失效日期" 
        :picker-options="pickerOptionEndDate">
    </el-date-picker>

        var vm = new Vue({
        el: "#app",
        data() {
            return {
                startDate: null,
                endDate: null,
                pickerOptionStartDate: {
                    disabledDate: (time) => {
                        if (this.endDate != null) {
                            return time.getTime() > new Date(this.endDate).getTime()
                        } else {
                            return false;
                        }
                    }
                },
                pickerOptionEndDate: {
                    disabledDate: (time) => {
                        return time.getTime() < new Date(this.startDate).getTime();
                    }
                }
            }
        }
    })

    在选择开始日期是我们先要判断用户是否先选择了失效日期，如果用户选择了失效日期，那么我们的生效日期就要比失效日期要小。而我们这里的startDate和endDate用的都是字符串，所以要使用new Date将字符串时间转成Date，再转成时间戳去比较。

    还需要注意一点的是，startDate要设置为null而不能设置为空字符串，因为在日期框清除当前的时间时，会赋值给相应字符串为null，而不是空字符串，而且空字符串和null不相等，所以在比较上会出现错误。如果不设置为null，就要把判断语句设置的能够正常判断。
